### 2b. Key Components

#### `wait_for_user_file`
**Purpose:** Blocks execution until the user records and saves their speech to the run directory in any supported audio format (`.mp3`, `.m4a`, `.wav`).
```python
def wait_for_user_file(base_name: str) -> tuple[str, str]:
    supported_extensions = [".mp3", ".m4a", ".wav"]
    while True:
        input("Press [Enter] once you have saved the file to continue...")
        for ext in supported_extensions:
            full_path = os.path.join(run_log_dir, f"{base_name}{ext}")
            if os.path.exists(full_path):
                return full_path, ext[1:]
```

#### `transcribe_speech`
**Purpose:** Sends base64-encoded audio responses (e.g. from the operator) directly to OpenRouter's `google/gemini-2.5-flash` model for high-accuracy Polish transcription.
```python
def transcribe_speech(audio_base64: str, audio_format: str) -> str:
    url = "https://openrouter.ai/api/v1/chat/completions"
    payload = {
        "model": "google/gemini-2.5-flash",
        "messages": [
            {
                "role": "user",
                "content": [
                    {"type": "text", "text": "Transcribe this audio recording in Polish. Write down every single word exactly as spoken, without summarizing."},
                    {"type": "input_audio", "input_audio": {"data": audio_base64, "format": audio_format.lower()}}
                ]
            }
        ]
    }
```

## 3. 🧱 Main Struggles & How They Were Resolved

- **Problem**: API returned `401 Unauthorized` for OpenRouter's text-to-speech `/api/v1/audio/speech` endpoint when trying to generate synthetic voice using `openai/tts-1`.
  - **Root Cause**: OpenRouter does not support standard OpenAI TTS endpoints for common keys, and synthetic robotic voices generated by local gTTS libraries were flagged as robotic by the operator, triggering `400 Bad Request` ("Mowisz w bardzo dziwny sposob...").
  - **Resolution**: Refactored the architecture to be **Human-in-the-Loop**, where the user records their own real voice (using M4A/MP3), and the agent handles the API postings, file transcription, and script generation.
  - **Takeaway**: Synthetic text-to-speech is easily detected by advanced voice security systems. Leveraging real human voice recordings guarantees verification.

- **Problem**: The dialogue got burned/reset because turns were out of order.
  - **Root Cause**: The operator expects a strict Turn-by-Turn state protocol. We combined the introduction and the inquiry in Turn 1, leading to an identity confirmation but subsequent failures.
  - **Resolution**: Rebuilt the flow into a strict sequence: Turn 1 (Introduction only) $\rightarrow$ Turn 2 (Road Inquiry explaining the Zygfryd secret transport) $\rightarrow$ Turn 3 (Requesting deactivation, justifying with Zygfryd's orders and citing the password `BARBAKAN`).

## 5. 💡 Agentic Patterns Observed

- **Human-in-the-Loop**: The core of the final working solution. The user acted as the speech executioner while the agent acted as the brain, parsing responses and driving the state machine.
- **Multimodal Classification**: Used `google/gemini-2.5-flash` with the `input_audio` modality schema to transcribe audio.

## 6. 🔁 What Would You Do Differently

- We would skip trying to use standard OpenAI `/v1/audio/speech` endpoints on OpenRouter entirely.
- We would immediately prototype an interactive human-guided recording agent since real voice bypasses anti-bot verification instantly.

---

## 7. 🧠 Key Learnings

> **Robotic Detection:** Operators and systems in AI_Devs tasks inspect voice naturalness. Local TTS packages like `gTTS` generate robotic sound signatures that fail security checks.
> **OpenRouter Audio:** Standard OpenAI audio endpoints are not supported by default. Multimodal input payloads (`type: "input_audio"`) on `google/gemini-2.5-flash` or `google/gemini-1.5-pro` are the recommended way to transcribe audio.
> **Dynamic Fallbacks:** Building interactive manual overrides inside CLI scripts ensures that dialogue flows don't crash when transcription outputs are slightly ambiguous.
